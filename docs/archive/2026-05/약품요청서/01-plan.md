# Plan — 약품요청서(잉크 발주 요청서) 페이지

작성일: 2026-05-24
작성자: 김선명(생산관리팀) — 단일 사용자 운영
관련 메모리: [[project-overview]] [[project-data-model]]

## 1. 배경 / Problem

`docs/RUNBOOK.md`와 도메인 메모리(`project-overview.md`)에는 약품요청서가 "주간 사출계획에서 자동 산출되는 산출물"로 명시돼 있다. 하지만 현재 NAV(`app.jsx`)에는 약품요청서를 출력·공유할 수 있는 페이지가 없다.

현장은 매일 사출계획에 따라 어떤 잉크를 얼마만큼 써야 하는지 발주(요청)해야 한다. 지금은 `넣어줄 잉크`(ink-add) 페이지에서 "오늘 주/야 + 내일 주" 3시프트만 보여주고, 잉크 수만 카운트한다. **공식 발주 양식이 없다.**

## 2. 사용자 시나리오

> 김선명(생산관리팀)이 매일 아침/저녁 또는 주 시작 시점에 다음을 한다.
>
> 1. 좌측 NAV `현장 공급 > 약품요청서` 클릭
> 2. 기본은 "이번 주(월~일)" 범위. 필요시 preset(오늘/이번주/다음주/사용자 지정)으로 변경
> 3. 화면에 잉크별 발주 표가 자동 산출됨
>    - 품목코드 / 잉크명 / 호기 / 사용 횟수(세트 단위) / 비고
> 4. 우상단 `인쇄` 클릭 → 브라우저 인쇄 다이얼로그 → A4 한 장 또는 PDF로 저장
> 5. 출력물에는 작성일, 작성자(김선명), 범위, 합계 포함

## 3. 도메인 규칙 (확정)

- **세트(set) 단위**: 1호기 × 1시프트 = 1세트 (메모리 `project-overview` 기준)
- **잉크 추출 흐름**: 사출계획 셀(`data.injection[층][i].schedule[요일][shift]`) → 제품명 → `data.products`에서 inks 배열(1·2·3도) → 잉크별 카운트
- **품목코드 lookup**: `data.machineAssignments` 에서 `ink → code` 매핑 (1잉크 = 1코드, 메모리 `project-data-model` 기준)
- **호기 정보**: `machineAssignments`의 `machine` 필드. 같은 잉크가 여러 호기에 잡힐 일은 없지만(1잉크=1호기) 안전하게 사출계획상 실제 사용 호기를 표시
- **층 구분**: `data.injection` 키가 `3층`/`1층`. 3F = 10~27,40~43호기 / 1F = 44~59호기
- **빈 셀**: 무시 (집계에 포함 안 함)
- **잉크 없는 제품**: 무시 + 화면 하단에 "잉크 미등록 제품 N건" 경고

## 4. 입력 / 출력

### 입력
- `data.injection` (사출계획)
- `data.products` (제품 → 잉크)
- `data.machineAssignments` (잉크 → 품목코드, 호기)
- 사용자: 날짜 범위(요일 시작~끝), 시프트 필터(주/야/전체), 층 필터(3F/1F/전체)

### 출력(화면)
한 줄 = 잉크 1종:
| 품목코드 | 잉크명 | 사용 호기 | 3층 세트 | 1층 세트 | 총 세트 | 비고 |

합계: 총 잉크 종류 수, 총 세트 수, 3F/1F 분리 합계.

### 출력(인쇄 PDF)
- A4 portrait, 헤더: "잉크 발주 요청서"
- 메타: 작성일 / 작성자 / 범위 / 작성 시각
- 본문 표: 위와 동일
- 푸터: 합계 + "본 요청서는 자동 산출되었습니다" 안내

## 5. 범위 (Scope)

### IN
- 새 페이지 `pages/chemicals.jsx` (이름: `ChemicalsPage`, NAV id: `chemicals`, 라벨: `약품요청서`)
- 날짜 범위 / 시프트 / 층 필터
- 인쇄용 CSS (`@media print`)
- `data-service.js`에 순수 집계 함수 `aggregateChemicalRequest(data, opts)` 추가
- 단위 테스트 (tests/data-service.test.js)

### OUT (이번 PDCA에서 안 함)
- 약품 함수율(38/43/55%) 등급 표시 — 데이터 모델에 없음. 추후 등급 필드가 추가되면 별도 PDCA.
- 발주 이력 보관(누가 언제 출력했는지) — 다음 단계 검토
- 이메일/메신저로 자동 전송 — Out
- 잉크 외 부자재 발주(컨테이너, 라벨 등) — Out

## 6. 성공 기준

1. 사출계획에 100건 이상 셀이 있을 때 1초 이내 집계 표시
2. 인쇄 미리보기에서 A4 1~2장에 깔끔히 들어감 (잉크 50종 기준)
3. 품목코드 없는 잉크는 표에서 "미입력" 배지로 표시 → 사용자가 마스터를 보완할 수 있게
4. 단위 테스트 3+개 통과 (빈 데이터, 정상 케이스, 코드 미입력 케이스)
5. 기존 페이지(`InkAddPage`) 동작 영향 없음

## 7. 위험 / 미정 사항

- **약품 등급 데이터 부재**: 메모리에서는 "38/43/55% 함수율"이 호기별로 있다고 했지만 실제 `machineAssignments`에는 없음. → 이번 PDCA 후 [[project-data-model]] 메모리 갱신 필요.
- **요일 키 한계**: 사출계획은 `{월,화,수,목,금,토,일,차주월}` 요일 키만 있어서, 과거 N주 회고가 불가능. → [[project-history-view]]의 Option B 결정 후 가능.
- 단일 사용자 운영이라 권한/감사 로그는 일단 안 다룸.
