# 잉크/사출 생산계획 실행 및 백업 안내

## 실사용 PC 실행

1. `잉크 재고 시스템.exe`를 더블클릭합니다.
2. 브라우저가 자동으로 열리면 사용합니다.
3. 검은 터미널 창은 뜨지 않고, 우측 하단 트레이에 아이콘이 유지됩니다.

서버는 PC 내부 주소 `http://127.0.0.1:8765/`에서만 동작합니다.

브라우저를 닫아도 트레이 앱이 켜져 있으면 저장은 계속 동작합니다. PC를 끄거나 트레이 메뉴에서 `종료`를 누르면 서버가 종료됩니다.

개발 PC에서는 `start.vbs`로도 실행할 수 있지만, 실사용 PC에는 Python이 없으므로 `.exe` 배포본을 사용합니다. 기존처럼 `python -m http.server`로 실행하면 화면은 열리지만 파일 DB 저장과 백업 API가 동작하지 않습니다.

## 트레이 메뉴

작업표시줄 우측 하단 아이콘을 우클릭하면 아래 기능을 사용할 수 있습니다.

- `열기`: 브라우저에서 앱을 엽니다.
- `지금 백업`: 현재 데이터를 즉시 백업합니다.
- `백업 폴더 열기`: 백업 파일 폴더를 엽니다.
- `자동 백업 등록 (18:30)`: 이 PC에 매일 18:30 자동 백업을 등록합니다.
- `자동 백업 해제`: 등록된 자동 백업을 해제합니다.
- `종료`: 앱 서버와 트레이 아이콘을 종료합니다.

## API 키

오른쪽 위 `API 키 필요` 버튼을 누르고 Gemini API 키를 한 번 저장합니다.

저장 후에는 이 PC에서 앱을 다시 실행해도 자동으로 기억됩니다. API 키를 바꾸거나 지우려면 같은 설정 창에서 수정하거나 `키 삭제`를 누릅니다.

API 키는 이 배포 폴더 안의 `data/settings.json`에 저장됩니다. 이 폴더는 담당자 PC 내부에서만 관리하고, 외부 공유 폴더나 메신저로 보내지 않습니다.

## 데이터 저장 위치

- 현재 데이터: `data/db/current.json`
- 자동/수동 백업: `data/backups/*.json`
- 앱 설정: `data/settings.json`
- 초기 데이터: `data/clean.json`

저장은 임시 파일에 먼저 쓴 뒤 `current.json`으로 교체합니다. 갑작스러운 정전 중 파일이 반쯤 써지는 위험을 줄이기 위한 방식입니다.

## 자동 백업

자동 백업 시간은 매일 `18:30`입니다.

실사용 PC에서 처음 한 번만 트레이 메뉴의 `자동 백업 등록 (18:30)`을 누릅니다.

작업 스케줄러 이름은 `InkPlanDailyBackup`입니다.

## 오프사이트 백업 (권장 — PC 고장 대비)

로컬 백업은 전부 이 PC 안(`data/backups/`)에 있으므로, PC가 고장나면 함께 사라집니다.
매일 백업 뒤 외부 위치(네트워크 드라이브·클라우드 동기화 폴더·USB)로 자동 미러링하려면
**목적지 한 곳만 지정**하면 됩니다. 둘 중 하나로 설정:

1. 환경변수 — 시스템 환경변수에 `INK_PLAN_OFFSITE_DIR` 추가 (예: `D:\ink_plan_백업` 또는 `\\NAS\backup`)
2. 설정 파일 — `data/settings.json`에 `"offsiteBackupDir": "D:\\ink_plan_백업"` 한 줄 추가

지정하면 매일 18:30 백업 직후 자동으로 `<목적지>/ink_plan/`에 `current.json`과 최근 백업(최대 60개)이
증분 복사됩니다. **미지정이거나 드라이브 미연결이면 조용히 건너뛰고 주 백업은 정상 동작**합니다.

- 수동 실행/점검: `python scripts/offsite_backup.py` (또는 배포본에서 동일 스크립트)
- 동작 기록: `data/logs/offsite.log`
- 클라우드를 쓰면(예: OneDrive/드라이브 동기화 폴더를 목적지로 지정) 사실상 오프사이트 자동 보관이 됩니다.

## 수동 백업

앱 우측 상단 설정 버튼을 열고 `데이터 백업 > 지금 백업`을 누릅니다.

앱을 열 수 없는 상황에서는 `backup-now.vbs`를 더블클릭하면 즉시 백업합니다.

실사용 `.exe` 배포본에서는 앱을 열 수 없는 상황에도 `잉크 재고 시스템.exe --backup-now`로 백업할 수 있습니다. 자동 백업은 이 명령을 사용합니다.

## 복구

앱 우측 상단 설정 버튼을 열면 최근 백업 목록이 보입니다. 원하는 백업의 `복원` 버튼을 누르면 현재 데이터는 `before_restore` 백업으로 먼저 보관된 뒤 선택한 백업으로 복원됩니다.

앱을 열 수 없는 상황에서는 파일 단위로 복구할 수 있습니다.

1. `data/backups`에서 복구할 날짜의 `.json` 파일을 고릅니다.
2. 앱을 닫습니다.
3. 현재 `data/db/current.json`을 따로 복사해 둡니다.
4. 선택한 백업 파일을 `data/db/current.json`으로 복사합니다.
5. `잉크 재고 시스템.exe`로 다시 실행합니다.

## 배포본 만들기

개발 PC에서 아래 명령으로 Python 없는 PC용 배포 폴더를 만듭니다.

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_release.ps1
```

완료 후 `release/잉크 재고 시스템/` 폴더를 실사용 PC로 복사합니다. 사용자는 그 안의 `잉크 재고 시스템.exe`만 실행하면 됩니다.

## 사내망 서버 PC로 운영 (여러 PC가 같은 데이터 공유)

서버 PC 한 대를 두고, 다른 PC들은 브라우저로 `http://서버IP:8766` 에 접속해 **같은 데이터**를 씁니다.
동시편집은 자동으로 보호됩니다(OCC). 사내망(신뢰망) 전용입니다.

### 서버 PC 최초 세팅 (한 번만)
1. **프로젝트 폴더째 복사**로 서버 PC에 옮깁니다(USB/공유폴더).
   - ⚠️ `git clone` 은 `data/db/` 가 gitignore 라 **DB가 안 딸려옵니다**. 반드시 **폴더 복사**로 옮겨야 현재 데이터(`data/db/current.json`)가 그대로 넘어갑니다. → **DB 재세팅 불필요.**
2. 서버 PC에 **Python 설치**(1회). git 이 있으면 이후 코드 업데이트가 자동화됩니다.
3. **고정 IP 권장**: 공유기에서 서버 PC에 IP 고정(예약). 안 하면 주소가 바뀝니다.
4. **Windows 방화벽**에서 `8766` 포트 **인바운드 허용**:
   ```
   netsh advfirewall firewall add rule name="InkPlan 8766" dir=in action=allow protocol=TCP localport=8766
   ```

### 서버 실행
서버 PC에서 **`서버-실행.bat`** 더블클릭. 이 스크립트가:
- `git pull` 로 **코드만 최신화**(DB는 `data/db/` = gitignore 라 절대 안 건드림)
- LAN 모드(`INK_PLAN_BIND=0.0.0.0`)로 서버 기동 → 창에 **다른 PC 접속 주소**(`http://서버IP:8766/`)를 표시
- 서버가 죽어도 **자동 재시작**

부팅 시 자동 실행하려면 **작업 스케줄러**에 "로그온 시 `서버-실행.bat` 실행"을 등록합니다.

### 다른 PC(클라이언트)
- 브라우저에서 **`http://서버IP:8766`** 접속만 하면 됩니다(설치 불필요).
- 서버 창에 표시된 주소를 그대로 쓰면 됩니다.

### 업데이트 철칙
- 코드(`.jsx`/`.py`)와 DB(`data/db/`)는 분리돼 있습니다. **업데이트는 `서버-실행.bat`(git pull)로 코드만** 갱신되고 `data/db/` 는 보존됩니다. 수동으로 `data/db/` 를 덮어쓰지 마세요.

### 참고 — API 키
OCR은 브라우저가 Gemini를 직접 호출하므로, 각 클라이언트 브라우저가 서버에서 API 키를 받아갑니다(사내망에서 키 1개 공유). 키를 서버 밖으로 내보내지 않으려면 서버측 OCR 프록시가 필요합니다(후속 과제).

### 로컬 단독 사용은 그대로
`INK_PLAN_BIND` 를 설정하지 않으면 기존처럼 **이 PC(127.0.0.1)에서만** 동작합니다. LAN 개방은 서버 PC에서 위 bat로 켤 때만 적용됩니다.
